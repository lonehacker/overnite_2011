Instructions


Before You Start

Welcome to the practice round of ACM-ICPC Overnite '11 Programming Contest. Before you start the contest, please be aware of the following notes:

  • There are seven problems in this contest.
  • Any of your solutions can use any of the languages C, C++ or Java. Use only the compilers gcc, g++, and javac.
  • Only follwing headers should be used:
    • For C: stdio.h, stdlib.h, string.h, math.h
    • For C++: cstdio, cstdlib, cmath, cstring, vector, iostream, string, sstream, complex.
    • For Java: java.lang.*, java.util.*.,
    • Don't use any 'system' calls.
  • All solutions must read from standard input and write to standard output. In C this is scanf/printf, in C++ this is cin/cout, and in Java this is System.in/out.
  • The judges will ignore all output sent to standard error. From your workstation you may test your program with an input file by redirecting input from a file:
    program < file.in
    or in Java:  java Program < file.in
  • Solutions for problems submitted for judging are called runs. Each run will be judged.
    Refresh the Score card block to view the status of the submission.
  • This problem set contains sample input and output for each problem. However, you may be assured that the judges will test your submission against several other more complex datasets. Several questions require an efficient rather than naive solution.
    • If the response is "wrong", your program output was wrong.
    • If the response is "Time-Limit Exceeded", chances are that you need to implement a more efficient algorithm.
    • If the response is "E", your program contains something that is not admissible to run.
  • Runs for each particular problem will be judged in the order they are received. However, it is possible that runs for different problems may be judged out of order. For example, you may submit a run for B followed by a run for C, but receive the response for C first.
  • All lines of program input and output should end with a newline character (\n, endl, or println()). At the end of total output (including all test cases) there should be a newline.
  • All input sets used by the judges will follow the input format specification found in the problem description. You do not need to test for input that violates the input format specified in the problem.
  • Best wishes for your team. Remember the Olympic ideal: Treasure your experience of participation and your moments of struggle!
